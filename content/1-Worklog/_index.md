**On this page**, you will need to introduce your worklog. **How** did you complete it? How many weeks did you take to complete the program? **What** did you do in those weeks?

Typically, and as a standard, a worklog is carried out over about 3 months (throughout the internship period) with weekly contents as follows:

### Internship Schedule (Weekly Worklog)

| Week | Topic & Objective | Detailed Content | Result & Milestone |
| :---: | :--- | :--- | :--- |
| **1** | **CULTURE ONBOARDING AND COST MANAGEMENT SETUP**<br>_Objective:_ Onboard into the work environment and build a financial security barrier for the AWS account. | <ul><li>**Monday (1.5h):** Understand the organizational structure and technical reporting standards.</li><li>**Tuesday (1h):** Initialize AWS account and get familiar with the Management Console.</li><li>**Wednesday (2h):** Research and set up AWS Budgets; configure cost threshold alerts via Email.</li><li>**Thursday (1.5h):** Install and configure AWS CLI on a personal computer to prepare for command-line management.</li><li>**Friday (1h):** Summarize main service groups (Compute, Storage, Database) in the Cloud environment.</li></ul> | _Completion Date:_ 09/01/2026<br><br>_Result:_ AWS account is ready; clearly understand the cost control process to avoid unexpected risks. |
| **2** | **IDENTITY MANAGEMENT (IAM) AND SECURITY**<br>_Objective:_ Implement security standards for users and delegate system access. | <ul><li>**Monday (1.5h):** Understand IAM theory: Users, Groups, and Roles.</li><li>**Tuesday (1.5h):** Practice creating a separate Admin account; configure Multi-Factor Authentication (MFA) for all accounts.</li><li>**Wednesday (2h):** Research IAM Policies JSON structure and practice writing Least Privilege policies.</li><li>**Thursday (1h):** Practice the Switch Role mechanism for flexible access management.</li><li>**Friday (1h):** Recheck security layers and perform cleanup of unused Users.</li></ul> | _Completion Date:_ 16/01/2026<br><br>_Result:_ Completed a secure multi-user management environment; mastered the "Zero Trust" security mindset. |
| **3** | **NETWORK FOUNDATION AND VPC (PART 1: INFRASTRUCTURE INITIALIZATION)**<br>_Objective:_ Build the backbone for the virtual private cloud and public subnets. | <ul><li>**Monday (2h):** Research IP addressing and CIDR calculation for a large VPC system.</li><li>**Tuesday (1.5h):** Initialize VPC and set up Public Subnets according to standard models.</li><li>**Wednesday (1.5h):** Configure Internet Gateway (IGW) and attach it to the VPC.</li><li>**Thursday (1.5h):** Set up Route Tables for Public Subnets to allow outbound internet traffic.</li><li>**Friday (1.5h):** Test basic network connectivity by assigning static IPs to resources.</li></ul> | _Completion Date:_ 23/01/2026<br><br>_Result:_ Successfully deployed basic network infrastructure capable of communicating with the external environment. |
| **4** | **NETWORK FOUNDATION AND VPC (PART 2: ISOLATION AND SECURITY)**<br>_Objective:_ Establish a private network zone and deep traffic control mechanisms. | <ul><li>**Monday (1.5h):** Deploy Private Subnets to host sensitive resources (Database, Backend).</li><li>**Tuesday (2h):** Configure NAT Gateway to allow one-way Internet access for Private Subnets.</li><li>**Wednesday (1.5h):** Differentiate and practice configuring Security Groups and Network ACLs.</li><li>**Thursday (1.5h):** Set up a Bastion Host (Jump Box) for secure access to servers in Private Subnets.</li><li>**Friday (1h):** Summarize the Multi-tier Architecture network diagram.</li></ul> | _Completion Date:_ 30/01/2026<br><br>_Result:_ Completely built a secure network environment, ensuring privacy for internal data. |
| **5** | **OBJECT STORAGE SERVICE (AMAZON S3)**<br>_Objective:_ Manage unstructured data and optimize long-term storage costs. | <ul><li>**Monday (1h):** Create S3 Buckets and learn about naming conventions and storage regions.</li><li>**Tuesday (1.5h):** Practice uploading data, metadata management, and Versioning configuration.</li><li>**Wednesday (1.5h):** Set up Lifecycle Policies to automatically transition data to S3 Glacier.</li><li>**Thursday (2h):** Configure S3 Bucket Policies and ACLs to control public/internal access.</li><li>**Friday (1h):** Experiment with static website hosting directly on S3.</li></ul> | _Completion Date:_ 06/02/2026<br><br>_Result:_ Mastered the data lifecycle management process and optimized Cloud storage budget. |
| **6** | **CLOUD COMPUTING AND BLOCK STORAGE (EC2 & EBS)**<br>_Objective:_ Deploy and operate virtual servers with a durable data backup mechanism. | <ul><li>**Monday (1.5h):** Initialize EC2 instances (Linux); learn how to choose the right Instance Type.</li><li>**Tuesday (1.5h):** Manage EBS Volumes: Practice attaching extra disks and resizing capacity.</li><li>**Wednesday (2h):** Deploy backup mechanisms using EBS Snapshots and learn data recovery.</li><li>**Thursday (1.5h):** Use User Data to automate Web Server (Apache/Nginx) installation on boot.</li><li>**Friday (1h):** Practice managing server states (Start/Stop/Terminate) and track costs.</li></ul> | _Completion Date:_ 13/02/2026<br><br>_Result:_ Successfully operated virtual servers; mastered storage resource management skills. |
| **7** | **RELATIONAL DATABASE (AMAZON RDS) - PART 1**<br>_Objective:_ Deploy and connect a relational database management system in the cloud environment. | <ul><li>**Monday (2h):** Create DB Subnet Groups and prepare the network environment for databases.</li><li>**Tuesday (1.5h):** Initialize an RDS MySQL instance in the Private Subnet zone.</li><li>**Wednesday (1.5h):** Set up Security Groups to allow connections from EC2 servers (Web tier).</li><li>**Thursday (1h):** Configure Parameter Groups and Option Groups to optimize parameters.</li><li>**Friday (1.5h):** Practice connecting to the database via applications and test data retrieval.</li></ul> | _Completion Date:_ 20/02/2026<br><br>_Result:_ Completed a stable and secure 2-tier application model (Web-Database). |
| **8** | **ADVANCED RDS AND SYSTEM BACKUP**<br>_Objective:_ Enhance high availability and establish data protection workflows for the business. | <ul><li>**Monday (1.5h):** Research Automated Backups and Retention Periods in RDS.</li><li>**Tuesday (1.5h):** Practice creating manual DB Snapshots and Point-in-Time Recovery (PITR).</li><li>**Wednesday (2h):** Configure Multi-AZ Deployments to ensure automatic recovery during failures.</li><li>**Thursday (1.5h):** Initialize Read Replicas to offload traffic for heavy read queries.</li><li>**Friday (1h):** Summarize critical data protection strategies on the Cloud.</li></ul> | _Completion Date:_ 27/02/2026<br><br>_Result:_ Mastered professional database operations with high availability and fault tolerance. |
| **9** | **ELASTIC LOAD BALANCING (ELB) AND TRAFFIC DISTRIBUTION**<br>_Objective:_ Build a system capable of intelligently distributing traffic to multiple servers. | <ul><li>**Monday (1.5h):** Learn about Load Balancer types (Application, Network, Gateway) and use cases.</li><li>**Tuesday (1.5h):** Initialize an Application Load Balancer (ALB) and set up Target Groups.</li><li>**Wednesday (2h):** Practice configuring Health Checks for Load Balancers.</li><li>**Thursday (1.5h):** Configure Listener Rules for Path-based routing.</li><li>**Friday (1h):** Test load balancing capabilities by accessing the application via ALB's DNS.</li></ul> | _Completion Date:_ 06/03/2026<br><br>_Result:_ The system has better load tolerance and avoids disruption during single-server failures. |
| **10**| **AUTO SCALING ARCHITECTURE**<br>_Objective:_ Automatically adjust the number of servers based on actual user demand. | <ul><li>**Monday (2h):** Create Launch Templates to define the baseline server configuration.</li><li>**Tuesday (1.5h):** Initialize an Auto Scaling Group (ASG) integrated with the ALB.</li><li>**Wednesday (1.5h):** Set up Scaling Policies based on CPU utilization or Request count metrics.</li><li>**Thursday (1.5h):** Practice "Self-healing" checks: Terminate a server and observe ASG's response.</li><li>**Friday (1h):** Optimize costs by configuring minimum and maximum server limits.</li></ul> | _Completion Date:_ 13/03/2026<br><br>_Result:_ Successfully built an architecture capable of elastic scaling, optimizing both performance and cost. |
| **11**| **MONITORING AND PERFORMANCE MANAGEMENT (CLOUDWATCH)**<br>_Objective:_ Comprehensively observe system operations and set up automated incident response. | <ul><li>**Monday (1.5h):** Learn about CloudWatch Metrics: Track important metrics like CPU, Disk I/O, Network.</li><li>**Tuesday (1.5h):** Set up CloudWatch Alarms to send SNS notifications on high load.</li><li>**Wednesday (2h):** Build a centralized Dashboard for a holistic view of project health.</li><li>**Thursday (1.5h):** Configure CloudWatch Logs to collect error logs from EC2 servers.</li><li>**Friday (1h):** Recheck incident response workflows based on the configured alarms.</li></ul> | _Completion Date:_ 20/03/2026<br><br>_Result:_ Mastered monitoring tools to help administrators proactively maintain system stability. |
| **12**| **ARCHITECTURE REVIEW AND FINAL OPTIMIZATION**<br>_Objective:_ Re-evaluate the entire system against professional standards and cleanup resources. | <ul><li>**Monday (2h):** Evaluate the system based on the 5 pillars of the AWS Well-Architected Framework.</li><li>**Tuesday (1.5h):** Re-optimize Security Groups and IAM Policies (final review process).</li><li>**Wednesday (1.5h):** Do a comprehensive check of incurred costs and create a financial report.</li><li>**Thursday (2h):** Execute cleanup procedures: Delete NAT Gateways, ALBs, EIPs, and terminate servers.</li><li>**Friday (1h):** Summarize lessons learned and complete the final internship report.</li></ul> | _Completion Date:_ 27/03/2026<br><br>_Result:_ Finished the internship with solid foundational knowledge; the system is appropriately packed and cleaned. |
