### Internship Schedule (Weekly Worklog)

- [Week 1: Culture Onboarding and Cost Management Setup. Onboard into the work environment and build a financial security barrier for the AWS account.](1.1-Week1)
- [Week 2: Identity Management (IAM) and Security. Implement security standards for users and delegate system access.](1.2-Week2)
- [Week 3: Network Foundation and VPC (Part 1: Infrastructure Initialization). Build the backbone for the virtual private cloud and public subnets.](1.3-Week3)
- [Week 4: Network Foundation and VPC (Part 2: Isolation and Security). Establish a private network zone and deep traffic control mechanisms.](1.4-Week4)
- [Week 5: Object Storage Service (Amazon S3). Manage unstructured data and optimize long-term storage costs.](1.5-Week5)
- [Week 6: Cloud Computing and Block Storage (EC2 & EBS). Deploy and operate virtual servers with a durable data backup mechanism.](1.6-Week6)
- [Week 7: Relational Database (Amazon RDS) - Part 1. Deploy and connect a relational database management system in the cloud environment.](1.7-Week7)
- [Week 8: Advanced RDS and System Backup. Enhance high availability and establish data protection workflows for the business.](1.8-Week8)
- [Week 9: Elastic Load Balancing (ELB) and Traffic Distribution. Build a system capable of intelligently distributing traffic to multiple servers.](1.9-Week9)
- [Week 10: Auto Scaling Architecture. Automatically adjust the number of servers based on actual user demand.](1.10-Week10)
- [Week 11: Monitoring and Performance Management (CloudWatch). Comprehensively observe system operations and set up automated incident response.](1.11-Week11)
- [Week 12: Architecture Review and Final Optimization. Re-evaluate the entire system against professional standards and cleanup resources.](1.12-Week12)
